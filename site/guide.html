End-to-End Usage Tutorial

This is the full operator path for Mosaic: install, configure, run, connect channels, operate gateway, and keep daily diagnostics. Every command below is copy-runnable.

Recommended mode: use --project-state so all state stays inside your repo at .mosaic/.

1. Install and verify

# after installation
mosaic --version
mosaic --help
mosaic --project-state directory --ensure --check-writable

If mosaic is not found, add $HOME/.cargo/bin to your shell PATH.

2. Initialize one profile

export OPENAI_API_KEY=<your_key>

mosaic --project-state --profile default setup \
  --base-url https://api.openai.com \
  --api-key-env OPENAI_API_KEY \
  --model gpt-4o-mini

3. Multi-profile setup (OpenAI + Azure)

# OpenAI profile
mosaic --project-state --profile openai setup \
  --base-url https://api.openai.com \
  --api-key-env OPENAI_API_KEY \
  --model gpt-4o-mini

# Azure profile (must end with /openai/v1)
export AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
export AZURE_OPENAI_API_KEY="<your_key>"

mosaic --project-state --profile az-openai setup \
  --base-url "$AZURE_OPENAI_BASE_URL" \
  --api-key-env AZURE_OPENAI_API_KEY \
  --model gpt-4o-mini

4. Validate model connectivity

mosaic --project-state --profile az-openai models list
mosaic --project-state --profile az-openai models status
mosaic --project-state --profile az-openai models resolve
mosaic --project-state --profile az-openai health

5. Configure model aliases and fallback

mosaic --project-state --profile az-openai models aliases set fast gpt-4o-mini
mosaic --project-state --profile az-openai models aliases list

mosaic --project-state --profile az-openai models fallbacks add gpt-4.1-mini
mosaic --project-state --profile az-openai models fallbacks list

6. Run ask/chat/session workflow

# single turn
mosaic --project-state --profile az-openai ask "summarize this repo"

# multiline via file
mosaic --project-state --profile az-openai ask --prompt-file prompts/ask.txt

# interactive REPL
mosaic --project-state --profile az-openai chat
# in REPL: /help, /status, /new, /exit
# session management
mosaic --project-state session list
mosaic --project-state session show <session_id>
mosaic --project-state session resume <session_id>
mosaic --project-state --yes session clear

7. Configure channels (terminal/slack/telegram)

# terminal/local output channel
mosaic --project-state channels add --name local-terminal --kind terminal

# Slack webhook channel
mosaic --project-state channels add \
  --name team-slack \
  --kind slack_webhook \
  --endpoint https://hooks.slack.com/services/XXX/YYY/ZZZ

# Telegram bot channel
mosaic --project-state channels add \
  --name team-telegram \
  --kind telegram_bot \
  --chat-id <chat_id> \
  --token-env MOSAIC_TELEGRAM_BOT_TOKEN
# test + send + inspect
mosaic --project-state channels list
mosaic --project-state channels test <channel_id>
mosaic --project-state channels send <channel_id> --text "deployment started"
mosaic --project-state channels status
mosaic --project-state channels logs --channel <channel_id> --tail 30

8. Start and probe gateway

mosaic --project-state gateway install --host 127.0.0.1 --port 8787
mosaic --project-state gateway start
mosaic --project-state gateway status
mosaic --project-state gateway probe
mosaic --project-state gateway discover
mosaic --project-state gateway call status

# stop when needed
mosaic --project-state gateway stop

9. Apply execution policies (approvals + sandbox)

mosaic --project-state approvals get
mosaic --project-state approvals set confirm
mosaic --project-state approvals allowlist add "git status"

mosaic --project-state sandbox get
mosaic --project-state sandbox set standard
mosaic --project-state sandbox explain --profile elevated

10. Day-2 observability and diagnosis

mosaic --project-state status
mosaic --project-state health
mosaic --project-state doctor
mosaic --project-state dashboard

mosaic --project-state logs --tail 120
mosaic --project-state logs --follow
mosaic --project-state system presence
mosaic --project-state system event deploy.started --data '{"env":"prod"}'

11. Optional advanced modules

# memory
mosaic --project-state memory index --path .
mosaic --project-state memory search "channel retry" --limit 10
mosaic --project-state memory status

# security
mosaic --project-state security audit --path . --deep

# agents/plugins/skills
mosaic --project-state agents list
mosaic --project-state plugins list
mosaic --project-state skills list

12. JSON automation and maintenance commands

# machine-readable output
mosaic --project-state --json status
mosaic --project-state --json channels list
mosaic --project-state --json gateway probe

# maintenance
mosaic update
mosaic update --check

# destructive (requires explicit --yes)
mosaic --project-state --yes reset
mosaic --project-state --yes uninstall
Safety note: keep approvals=confirm and sandbox=standard in shared environments.

13. Regression checklist (quick)

  • setup succeeds for at least one profile.
  • models list, ask, and chat all run successfully.
  • channels add/test/send works for your primary channel.
  • gateway start/probe/call returns healthy results.
  • status/health/doctor/logs outputs are readable and complete.
